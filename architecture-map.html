Job = deferred / async work on a queue. Actions dispatch Jobs directly only for delayed, scheduled, chained, or batched work.
📦 Queue
Redis / database driver
⚙️ Worker
Horizon-monitored
🏃 Job runs handle()
  • • runs logic via Service / Model
  • • may fire its own Event(s)
  • • retries via $tries / $backoff
  • • final failure → failed()
Listeners & Jobs never catch inside handle() — the queue's retry mechanism owns failure.

Output → JsonResources
Writes → Actions

Complex reads → Queries
Actions are synchronous
External capabilities → Services
Listeners & Jobs handle async
Validation + auth → FormRequests
Events fire after commit
Transport → DTOs (no validation)
State machines for non-trivial status
Everything final by default
Model hooks = boot/booted, no logic
